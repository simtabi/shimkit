# v0.4.0 plan — three more tools (`env`, `gpg`, `logs`)

> Scope per maintainer (2026-05-15): continuation of the v0.3.0
> tool-extension cycle. Each fits the charter (host-machine
> dev-workflow, unix/linux/macos).

## Ordering

Increasing complexity, same as v0.3.0:

1. **`shimkit env`** (smallest, ~200 LOC manager + ~25 tests)
2. **`shimkit gpg`** (medium, ~300 LOC + ~35 tests)
3. **`shimkit logs`** (largest, ~250 LOC + ~30 tests; per-platform APIs)

Each lands as its own commit on `main`; cut v0.4.0 once all three
are in.

## Architecture rules — same five

1. All subprocess via `CommandRunner.run(...)` (argv-list, no shell).
2. All output via `UI.*`.
3. Config-driven values (defaults.json).
4. Builder pattern: `Manager.create().boot().run()`.
5. Fluent `self` returns.

## Per-tool surface

### `shimkit env` — `.env` file viewer + scaffolder

```text
shimkit env                              # interactive menu (list-only)
shimkit env show [PATH] [--reveal]       # cat .env with secrets redacted by default
shimkit env list                         # list all .env* files in the cwd tree
shimkit env scaffold PATH                # create from template; MODERATE prompt
shimkit env diff A B                     # show keys that differ between two files
shimkit env redact PATH                  # write a redacted copy of PATH
```

- **Redaction reuses `core/log.py::redact_value`** — same key-name
  matcher (`password|passwd|pwd|secret|token|api[_-]?key|authorization|key`)
  so secrets in `.env` redact the same way they do in JSONL logs.
- **`--reveal`** opt-in flag to show full values; default-deny.
- **No required root**; `.env` is per-user.
- **Pure parser** in `env/parser.py` (handles `KEY=value`,
  `KEY="quoted value"`, comments, blank lines, line continuations).
- **No extra deps** — stdlib only.

### `shimkit gpg` — GPG key + git-signing hygiene

```text
shimkit gpg keys list [--json]
shimkit gpg keys generate NAME [--type ed25519|rsa4096]    # MODERATE
shimkit gpg keys export KEY_ID                              # public-key export
shimkit gpg agent status [--json]                          # gpg-agent state
shimkit gpg git-signing show                               # current git signing config
shimkit gpg git-signing configure KEY_ID [--scope global|local]  # MODERATE
```

- **Shells out to `gpg`** for all key operations; same passphrase-
  isolation pattern as `ssh keys generate` (interactive `gpg
  --gen-key` reads passphrase directly from TTY).
- **`git-signing configure`** writes `user.signingkey` + `commit.gpgsign=true`
  via `git config --global` (or `--local` if specified). Reuses
  `CommandRunner`.
- **Requires `gpg` on PATH**; exits 69 if not.
- **No extra deps** — stdlib only.

### `shimkit logs` — system log tail/grep

```text
shimkit logs tail [--follow] [--lines N] [--predicate FILTER]
shimkit logs grep PATTERN [--since=1h] [--predicate FILTER]
shimkit logs system show [--limit N] [--predicate FILTER]
```

- **macOS**: shells out to `log show --predicate "..."` and `log stream`.
- **Linux**: shells out to `journalctl -u $unit -n $lines -f`.
- **Predicate syntax is per-platform** — passed through verbatim.
  Documented in `docs/tools/logs.md`.
- **JSON output**: per-line JSON records emitted via the existing
  `Event` model.
- **No mutations** — entirely read-only tool. No MODERATE prompts.
- **Requires `log` on macOS, `journalctl` on Linux**; exits 69 if absent.
- **No extra deps** — stdlib only.

## Test minimums (CONTRIBUTING.md baseline)

For each: `boot()` smoke, platform exit-69 (where applicable),
binary-missing exit-69 (for gpg + logs), every non-interactive
subcommand one happy + one sad path, `--json` parses, `--dry-run`
makes zero `CommandRunner.run` calls, MODERATE prompts blocked
under `--no-input`.

Mock at `CommandRunner.run` and `Platform.detect`; tmp_path fixtures
for any file-touching path.

## Out of scope this cycle

- **Cloud secrets managers** (Vault, AWS Secrets Manager, etc.) —
  out of charter.
- **Email signing / S/MIME** — separate concern from git-signing.
- **Centralised log aggregation** (rsyslog forwarder, OpenSearch) —
  out of charter; `shimkit logs` is for local triage.
- **YubiKey GPG cards** — vendor SDKs, revisit if there's demand.
- **Windows** — explicit charter exclusion.
