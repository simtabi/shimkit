# shimkit 0.5.0

User-facing summary of `0.5.0` — the ubuntu/ migration release. For
the full machine-readable changelog, see
[`CHANGELOG.md`](../../CHANGELOG.md).

---

## TL;DR

Migrated the legacy `ubuntu/` server-provisioning scripts into shimkit
under a **Docker-first** charter expansion. Three new sub-trees,
two new core primitives, eight new commands. 152 new tests.

```
shimkit db <engine>               container-first databases (5 engines)
shimkit stack lemp                multi-container LEMP recipe
shimkit web nginx vhost           hardened nginx vhost generator
shimkit shell colors              256-color palette diagnostic
```

The original 22-script ubuntu tree had **5 Critical + 7 High**
security flags from the audit — disabled apparmor, mongodb/mysql
binding to `0.0.0.0`, deprecated `apt-key adv`, `curl|sh` patterns.
The Docker-first design dissolves every Critical without per-tool
effort: shimkit's databases never bind wider than `127.0.0.1`,
never edit `/etc/`, and never call `apt-get` at runtime.

## What's new

### `shimkit db <engine>`

Container-first database orchestration. No host-install path —
shimkit's value here is the container lifecycle, not apt-key
wrangling.

Five engines: `mysql`, `mariadb`, `postgres`, `mongo`, `phpmyadmin`.
Each binds loopback-only by default, lives at
`~/.shimkit/data/db/<engine>-<id>/`, and carries a `shimkit.tool=db`
label so `docker-clean` can identify shimkit-managed containers.

```bash
shimkit db mysql up                                   # start a container
shimkit db mysql shell                                # interactive client
shimkit db mysql dump --out backup.sql                # dump to file
shimkit db mysql reset --confirm RESET-DB             # SEVERE
shimkit db ls                                         # list all shimkit dbs
```

Port conventions: mysql `:13306`, mariadb `:13307`, postgres
`:15432`, mongo `:17017`, phpmyadmin `:18080`. All loopback.

### `shimkit stack lemp`

Three-container LEMP recipe (db + php-fpm + nginx) on a per-project
user-defined bridge network so containers can `fastcgi_pass` each
other by name without exposing extra host ports. `$cwd` is
bind-mounted at `/srv/app`.

```bash
shimkit stack lemp up --project myapp
shimkit stack lemp exec --project myapp -- composer install
shimkit stack lemp logs --project myapp
shimkit stack lemp down --project myapp
shimkit stack ls
```

### `shimkit web nginx vhost`

Hardened nginx vhost generator. File-only by default; mutating the
host is SEVERE-tier with `--confirm APPLY-VHOST`.

```bash
shimkit web nginx vhost generate --name myapp --domain example.com \
    --root /var/www/myapp --flavor laravel --php-version 8.3
shimkit web nginx vhost apply --name myapp --source ./myapp.conf \
    --confirm APPLY-VHOST
```

Three flavors (static / php / laravel). Every flavor begins with
the `# managed-by: shimkit` marker so `apply` / `remove` refuse to
clobber non-shimkit vhosts. Modern defaults: `X-Frame-Options`,
`X-Content-Type-Options`, `Permissions-Policy: interest-cohort=()`,
`server_tokens off`, dotfile-deny, HSTS deliberately not included
(footgun on HTTP-only).

### `shimkit shell colors`

256-color ANSI palette diagnostic. Read-only — prints the basic 16,
the 6x6x6 cube, and the 24-step grayscale ramp, each cell labeled
with its index. Useful when a new terminal theme makes shimkit's
help output illegible and you want to see what your terminal
actually paints.

## Core additions

Two new primitives in `shimkit.core`:

### `shimkit.core.docker.DockerEnv`

Shimkit chokepoint for the docker-py SDK. Boots the daemon once,
verifies reachability, then issues every SDK call through one
class. Standardises container naming (`shimkit-<scope>-<kind>-<id>`)
and volume layout (`~/.shimkit/data/<scope>/`). Refuses to delete
anything outside `~/.shimkit/data/` via a `Path.relative_to` guard.

### `shimkit.core.version`

Tool-version detection + constraint enforcement. One source of
truth (the JSON `tools.versions` registry) consulted at three
enforcement points: install-time docs, runtime preflight,
`shimkit doctor`. Four distinct outcomes: `OK` / `OUT_OF_RANGE` /
`MISSING` / `UNPARSEABLE`. Five built-in detectors: docker, nginx,
git, gpg, python.

`shimkit doctor` now ends with a versions table:

```
versions
  docker     27.0.0     ok
  nginx      1.27.0     ok
  git        2.39.3     ok
  gpg        2.4.5      ok
  python     3.13.1     ok
```

## Migration

The legacy `ubuntu/` source tree (22 scripts, 88 files) was
archived under `.design/archive/ubuntu-snapshot-2026-05-15.tar.gz`
and removed. SHA-256 of the archive recorded in
`.design/plans/validation-report.md`.

**15 source features were deliberately skipped** — broken scripts,
out-of-charter host-install patterns, dotfiles-territory shell
tweaks. Full breakdown in
`.design/plans/feature-gap-analysis.md`.

Four features were deferred to later v0.x releases (all now done):

| Feature | Lands in |
|---------|----------|
| Generic cron editor | v0.6.0 |
| Laravel helpers | v0.7.0 |
| TLS / certbot | v0.8.0 |
| `--on-host` mode for db | v0.9.0 |

## Upgrading

```bash
uv tool upgrade shimkit
pipx upgrade shimkit
brew upgrade simtabi/tap/shimkit
```

No config changes required — `tools.db`, `tools.stack`, `tools.web`,
`tools.versions` are new blocks with sensible defaults. Existing
`~/.config/shimkit/shimkit.json` continues to validate.
